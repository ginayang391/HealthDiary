## 📊 題目A 資料流向分析

```
使用者填表單
    ↓
Frontend (index.html) fetch()
    ↓
POST /health-logs → Express API
    ↓

    sleep_hours < 6?
    ├─ YES → steps < 4000?
    │        ├─ YES → 高風險
    │        └─ NO  → mood_score < 5? → 高/中
    └─ NO  → steps < 5000?
             ├─ YES → mood_score < 5? → 中/低
             └─ NO  → mood_score < 6? → 中/低
    ↓
寫入 SQLite health_logs（含 risk_level）
    ↓
GET /health-logs → 回傳全部紀錄給前端
GET /health-logs/risk → 回傳最新風險等級
POST /health-logs/ai-advice → 呼叫 OpenRouter → AI 健康建議
```

這是設計問題：`loadLogs()` 執行後，風險評估會顯示資料庫裡**日期最新的那筆**，不一定是你剛填的。

已修改 `public/index.html` 表單送出的 callback，在 API 回傳成功後立即執行：

```javascript
// 修改後的表單送出處理
const data = await res.json();
if (!res.ok) { showMsg(data.error); return; }
showMsg(`✅ 新增成功！風險等級：${data.risk_level}`, 'success');
// 立即用剛送出的那筆資料更新風險評估區塊
latestLog = data;
updateRiskDisplay(data);
e.target.reset();
document.getElementById('mood-display').textContent = '5';
loadLogs();
```

修好了。現在流程是：
1. 填好表單 → 送出
2. API 回傳這筆新紀錄（含 `risk_level`）
3. **風險評估區塊立即顯示你剛填的那筆分析結果**
